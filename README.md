# root-path
> 분기별로 목표를 정하고 매일 기록하며 성장하는 스터디

## 📖 스터디 소개
분기별로 각자의 목표를 설정하고, 그 과정을 Git으로 기록하며 함께 성장하는 스터디입니다.

[스터디 문서 사이트 바로가기](https://the-study-group.github.io/root-path/)

## 🎯 스터디 목표
- 각자의 목표를 향한 **꾸준한 기록**
- 장기적으로 성장하는 **지속 가능한 커뮤니티**
- Git/GitHub 사용법 익히기
- 함께 성장하는 동료들

## 💡 규칙
### 필수 규칙
1. **목표 설정**: 분기별 또는 월별로 구체적인 목표
2. **매일 기록**: 가능한 매일 무언가를 기록
3. **회고 작성**: 목표 기간 종료 후 반드시 회고 작성
4. **Git 사용**: 모든 기록은 Git commit으로 관리

### 권장 사항
- **Commit 메시지**: `[날짜] 간단한 설명` 형식 권장
- **파일명**: `YYYY-MM-DD-제목.md` 형식 권장
- **구체적 목표**: 측정 가능한 목표 설정 권장

### 유연한 규칙
- **완벽함보다 꾸준함**: 오늘 못했다면 다음 날 다시 시작
- **목표 변경 가능**: 목표 문서 수정
- **자유로운 구조**: 폴더 구조, 파일 분리 방식 자유
- **유연한 주기**: 분기 단위가 부담스럽다면 월 단위로 목표

## 📁 폴더 구조
```
root-path/
├── README.md
└── docs/                   (배포용)
    ├── index.md            (메인 페이지)
    ├── 2024-Q4/
    │   ├── index.md        (Q4 소개)
    │   ├── member1/
    │   │   ├── index.md    (목표)
    │   │   └── logs/       (기록)
    │   └── member2/
    │       ├── index.md
    │       └── logs/
    └── 2025-Q1/
        └── ...
```

### 폴더 구조 설명
- `docs/`: 배포 폴더
- `docs/index.md`: 스터디 메인 페이지
- `docs/{분기}/`: 각 분기별 폴더
- `docs/{분기}/{닉네임}/index.md`: 해당 분기 목표
- `docs/{분기}/{닉네임}/logs/`: 일일 기록 (구조는 자유)

## 🔄 Git 전략

### Commit 규칙
- **일일 커밋**: 매일 기록은 본인 브랜치(Fork)에 커밋
- **Commit 메시지**
  - `[날짜] 간단한 설명`
  - 예: `[2024-11-20] 프로젝트 기획 작성`
  - 예: `[2024-11-20] (코테) - 백준 1234번 풀이`

### PR 규칙
- **주 1회 PR**: 매주 토요일 또는 일요일에 한 주 동안의 기록을 PR
- **PR 제목**: `[닉네임] YYYY.MM.DD ~ YYYY.MM.DD 주간 기록`
  - 예: `[root] 2025.11.25 ~ 2025.12.01 주간 기록`
- **PR 내용**: 
  - 이번 주 요약
  - 목표 진행 상황
  - 공유하고 싶은 내용

### Branch 전략

**1. Fork & PR**
1. Repository Fork
2. 본인 Fork에서 매일 커밋
3. 주말(토/일)에 Pull Request
4. 리뷰 후 Merge

**2. Organization Member**
1. Organization member 초대
2. 본인 branch 생성 (`member/닉네임`)
3. 본인 브랜치에 매일 커밋
4. 주말(토/일)에 Main으로 Pull Request
5. 리뷰 후 Merge

## 📅 모임

### 모임 일정
- **시간**: 토요일 또는 일요일
- **형태**: 오프라인 또는 온라인
- **소요 시간**: 약 1~2시간

### 모임 내용
1. **한 주 회고**
   - 이번 주 한 일
   - 어려웠던 점
   - 다음 주 계획

2. **지식 공유**
   - 배운 것 공유
   - 막힌 부분 함께 고민
   - 추천 자료 공유

## 🚀 참여 방법

1. **Repository Fork** (또는 Organization member 초대)
   
2. **본인 폴더 생성**
```
   docs/현재-분기/본인닉네임/
```

3. **목표 작성**
   - `docs/{분기}/{닉네임}/index.md` 작성

4. **매일 기록**
   - 본인만의 방식으로 꾸준히
   - 오늘 못했으면 내일부터 다시 시작
